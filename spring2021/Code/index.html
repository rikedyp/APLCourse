    Importing, Exporting and Distributing Code

    We have already learned the basics of saving and loading workspaces. Here we address some more specific things like how to use other people's code, how to let other people use your code, and how to distribute your application to users.

    Installed Libraries

    Dyalog installations come with a suite of libraries with pre-written code utilities and examples.

    Find which ones are available to you now with )lib.

    You can copy code into the workspace in several ways:

    • Copy the entire library into the current namespace (use pcopy to ensure that existing names are not overwritten):

      )pcopy dfns

    • Copy selected functions into the current namespace: 

      )copy dfns cal date days
cal 2↑⎕ts

    • Copy the entire library into a specific namespace: 

      'dfns'⎕ns⍬
dfns.⎕cy'dfns'

    • Copy the selected functions into a specific namespace: 

      'dfns'⎕ns⍬
'cal' 'date' 'days'dfns.⎕cy'dfns'

      Version Warning

      In Dyalog version 12.1, you must use a matrix left argument to copy multiple names with ⎕CY:

      (⍪↑'cal' 'date' 'days')dfns.⎕cy'dfns'
      The leading ⍪ ensures that ⎕CY does not fail when there is an empty character vector instead of a nested vector of character vectors. If you are dynamically generating this list of names, ensure it is a character matrix when passed to ⎕CY.

    Alongside APLcart, the dfns library (not to be confused with the dfns construct) contains a large number of useful functions for tasks ranging from number theory, graphs and tree data structures and games and graphics.

    Experimenting with code

    The ]Peek user command allows you to try an expression as if it was executed in a saved workspace without having to copy the contents of that workspace.

          ]peek dfns cal 2021 7
      ]peek -?

    Version Warning

    ]WS.Peek is not available in Dyalog version 12.1
